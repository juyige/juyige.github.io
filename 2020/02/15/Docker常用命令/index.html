Docker镜像操作

获取镜像

从镜像仓库获取镜像的命令docker pull

docker pull [选项] [Docker Registry 地址[:端口]/]仓库名[:标签]

运行镜像

docker run -it --rm ubuntu:16.04 bash

  • -i:交互式操作

  • -t:终端

  • –rm:容器退出后随之将其删除

  • ubuntu:16.04:这是指用ubuntu16.04镜像为基础来启动容器

  • bash:放在镜像名后的是命令，交互式shell

    exit退出容器

列出镜像

  • 列出已经下载的镜像

    docker image ls

  • 查看镜像、容器、数据卷所占用的空间

    docker system df

  • 显示虚悬镜像仓库名、标签均为的镜像称为虚悬镜像（dangling image）

    docker image ls -f dangling=true

  • 删除虚悬镜像（一般来说，虚悬镜像已经失去了价值，可以随意删除）

    docker image prune

删除本地镜像

  • 删除本地镜像

    docker image rm [选项] <镜像1>[<镜像1>···]

  • 批量删除镜像

    docker image rm $(docker image ls -q ubuntu) #删除所有仓库名为ubuntu的镜像

  • 删除所有在ubuntu:16.04之前的镜像

    docker image rm $(docker image ls -q -f before=ubuntu:16.04)

Docker容器操作

启动容器

启动容器的两种方式

  • 基于镜像新建一个容器并启动

  • 将在终止状态的容器重新启动

  1. 新建并启动

    docker run

    docker run ubuntu:16.04 /bin/echo 'hello world'

  2. 启动已经终止的容器

    docker container start 或者 docker start

    启动一个bash终端，允许用户进行交互

    docker run -t -i ubuntu:16.04 /bin/bash

    • -t让Docker分配一个伪终端并绑定到容器的标准输入上
    • -i让容器标准输入保持打开

后台运行

-d参数开启后台运行

docker run -d hello-world

停止运行的容器

  • 终止一个运行中的容器

    docker container stop

  • 查看终止住址状态的容器

    docker container ls -a

  • 重新启动终止状态的容器

    docker container start

  • 将一个运行态的容器终止，然后重新启动

    docker container restart

进入容器

docker exec命令可以进入到后台运行(-d参数)的容器中

docker exec -it 容器ID /bin/bash

导入和导出容器

  • 导出容器

    docker export 容器ID>导出文件名.tar

  • 导入容器

    使用docker import从容器快照文件中再导入为镜像

    cat 导出文件名.tar|docker import-镜像用户/镜像名:镜像版本

    通过指定的URL或者某个目录来导入

    docker import http://stydy.163.com/image.tgz example/imagerepo

删除容器

  • 删除容器

    1. 删除一个处于终止状态的容器

      docker container rm ubuntu:16.04

    2. 删除一个运行中的容器

      docker container rm -f ubuntu:16.04

  • 清理所有处于终止状态的容器

    docker container prune

Docker中最核心最基本的概念就是镜像（Image），容器（Container），以及仓库（Repository），其常用的操作命令也就是围绕这三个进行。
